# 🤖 Cline 核心指令集 (Strict Mode)

## 1. 核心哲学：原子化执行
- **参考信源**：行动需参考 `PROGRESS.md` 。
- **单步原则**：严禁同时推进两个 `[ ]` 任务。即使顺手能做的优化，也必须先完成当前任务并请示。
- **无感修改**：禁止重构或格式化与当前任务无关的代码块。

## 2. 标准操作流程 (SOP)
1. **任务定位**：读取 `PROGRESS.md`，确认第一个未勾选的 `[ ]` 任务。
2. **环境检查**：修改代码前，确保本地开发服务器（如 Django/FastAPI/C++ compiler）处于运行或就绪状态。
3. **精准编码**：仅修改为了实现该原子任务所必需修改的文件与代码行。
4. **闭环自测**：
   - **UI 任务**：必须使用 `puppeteer` 访问 `localhost` 检查控制台。
   - **逻辑/API 任务**：编写或运行现有的单元测试，或使用 `curl` 验证接口。
   - **错误处理**：若遇到报错，在尝试 2 次修复失败后立即停下手头工作，向用户汇报。
5. **同步与维护**：及时同步、更新`PROGRESS.md` 。任务通过后，同步到`PROGRESS.md` 将 `PROGRESS.md` 中对应的 `[ ]` 改为 `[x]`。

## 3. Token 极简协议
为了将上下文窗口留给核心代码，你必须遵守以下沟通禁令：
- **禁止废话**：严禁发送“好的”、“我明白了”、“准备开始”等无意义确认信息。
- **禁止解释**：除非代码极其晦涩或用户要求，否则严禁解释代码逻辑。
- **禁止回显**：严禁在对话框中粘贴长段代码。请直接利用工具操作文件。
